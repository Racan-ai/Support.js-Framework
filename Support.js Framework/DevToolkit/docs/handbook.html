1. Introduction & Overview

What is Support.js?

Support.js is a comprehensive, multi-framework JavaScript library designed to provide utility functions, components, and services for React, Angular, and Next.js applications. It offers a unified API across different frameworks, making it easier for developers to work with multiple technologies.

Key Features

üåê Multi-Framework Support

Works seamlessly with React, Angular, and Next.js applications

üì¶ Lightweight Bundle

Only 45KB minified with tree-shaking support

üîß Utility Functions

String, Date, and Number manipulation utilities

‚öõÔ∏è React Components

Pre-built hooks and components for React

üÖ∞Ô∏è Angular Services

Injectable services and interceptors

üì± TypeScript Ready

Full TypeScript support with type definitions

HTTP Interceptor

import { Injectable } from '@angular/core'; import { HttpInterceptor, HttpRequest, HttpHandler } from '@angular/common/http'; import { LoggerService } from 'support.js/angular'; @Injectable() export class SupportJsHttpInterceptor implements HttpInterceptor { constructor(private logger: LoggerService) {} intercept(req: HttpRequest<any>, next: HttpHandler) { // Log outgoing requests this.logger.debug(`HTTP ${req.method} ${req.url}`); const startTime = Date.now(); return next.handle(req).pipe( tap(event => { if (event instanceof HttpResponse) { const duration = Date.now() - startTime; this.logger.debug(`HTTP ${req.method} ${req.url} completed in ${duration}ms`); } }), catchError(error => { this.logger.error(`HTTP ${req.method} ${req.url} failed`, error); return throwError(error); }) ); } }

Data Processing Pipeline

import { stringUtils, dateUtils, numberUtils } from 'support.js'; class DataProcessor { constructor() { this.pipeline = []; } // Add processing steps addStep(stepFunction) { this.pipeline.push(stepFunction); return this; } // Process data through pipeline process(data) { return this.pipeline.reduce((result, step) => step(result), data); } // Predefined processing steps static steps = { // Clean and format names formatNames: (data) => { return data.map(item => ({ ...item, firstName: stringUtils.titleCase(item.firstName?.trim() || ''), lastName: stringUtils.titleCase(item.lastName?.trim() || '') })); }, // Format dates formatDates: (data) => { return data.map(item => ({ ...item, createdAt: dateUtils.formatDate(new Date(item.createdAt), 'YYYY-MM-DD'), updatedAt: dateUtils.formatDate(new Date(item.updatedAt), 'YYYY-MM-DD HH:mm') })); }, // Format currency values formatCurrency: (data) => { return data.map(item => ({ ...item, salary: numberUtils.formatCurrency(item.salary, 'USD'), bonus: numberUtils.formatCurrency(item.bonus || 0, 'USD') })); }, // Generate display names generateDisplayNames: (data) => { return data.map(item => ({ ...item, displayName: `${item.firstName} ${item.lastName}`, username: stringUtils.toKebabCase(`${item.firstName}-${item.lastName}`) })); } }; } // Usage example const processor = new DataProcessor() .addStep(DataProcessor.steps.formatNames) .addStep(DataProcessor.steps.formatDates) .addStep(DataProcessor.steps.formatCurrency) .addStep(DataProcessor.steps.generateDisplayNames); const rawData = [ { firstName: 'john', lastName: 'doe', salary: 50000, createdAt: '2024-01-15T10:30:00Z' } ]; const processedData = processor.process(rawData); console.log(processedData);

14. Migration Guide

From Version 0.9.x to 1.0.x

Breaking Changes

// v0.9.x import SupportJS from 'support.js'; SupportJS.string.camelCase('hello-world'); // v1.0.x import { stringUtils } from 'support.js'; stringUtils.toCamelCase('hello-world');

API Changes

v0.9.x v1.0.x Notes
string.camelCase() stringUtils.toCamelCase() Renamed for clarity
date.format() dateUtils.formatDate() Renamed for clarity
useStorage() useLocalStorage() More specific naming

Migration Script

# Run automated migration script npx support-js-migrate # Or manually update imports find src -name "*.js" -o -name "*.jsx" -o -name "*.ts" -o -name "*.tsx" | \ xargs sed -i 's/import SupportJS from '\''support\.js'\''/import { stringUtils, dateUtils } from '\''support.js'\''/g'

Step-by-Step Migration

// Step 1: Update package.json { "dependencies": { "support.js": "^1.0.0" // Update version } } // Step 2: Update imports // Old import SupportJS from 'support.js'; // New import { stringUtils, dateUtils, numberUtils } from 'support.js'; // Step 3: Update function calls // Old const result = SupportJS.string.camelCase('hello-world'); // New const result = stringUtils.toCamelCase('hello-world'); // Step 4: Update React hooks // Old const [value, setValue] = useStorage('key', 'default'); // New const [value, setValue] = useLocalStorage('key', 'default');
